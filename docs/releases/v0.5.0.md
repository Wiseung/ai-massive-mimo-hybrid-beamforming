# v0.5.0 Release Notes

## Scope

- Sionna-native channel extraction bridge
- `OFDMChannel(return_channel=True)`
- extracted `H_f = (B,Nsc,K,Nt)`
- native-channel-assisted + native-receiver-assisted
- optional Sionna dependency only
- no Sionna RT
- no ray tracing
- no 5G NR full stack

## New Features

- channel tensor audit
- `H_f` extraction utility
- axis validation
- native-channel-assisted beamforming chain
- extracted-H consistency quick benchmark
- extraction config sweep
- project-H_f vs extracted-H_f comparison
- minimal reproduction
- artifact manifest

## Headline Results

- `OFDMChannel(return_channel=True)` returns a channel tensor in the current Sionna 2.0.1 setup.
- Observed tensor layout is consistent with:
  `h = [B, rx, rx_ant, tx, tx_ant, ofdm_symbol, fft_bin]`.
- The current bridge extracts:
  `H_f = [B, Nsc, K, Nt]`.
- Axis spot-check passes with `max_abs_diff = 0.0`.
- `extraction_success = true`.
- `native_receiver_success = true`.
- Proxy/native rank agreement is low in the current quick consistency benchmark, so project proxy metrics should not replace native receiver metrics.

## Known Limitations

- not a full native-only benchmark
- precoder / learned model interface remains project-side bridge
- extracted-H consistency benchmark is quick / limited only
- no stable learned `> WMMSE-iter5` claim
- no Sionna RT
- no ray tracing
- no 5G NR full stack
- not production e2e
