# v0.7.0 Release Notes

## Scope

- CSI consumer unification
- `ExtractedCSI` as the preferred input interface
- analytic / learned / native-chain consumers support `ExtractedCSI`
- raw `H_f` retained as a backward-compatible fallback
- optional Sionna dependency only
- no Sionna RT
- no ray tracing
- no 5G NR full stack

## New Features

- CSI consumer audit
- CSI input adapter utilities
- analytic consumer unification
- learned consumer unification
- unified CSI consumer demo
- unified-vs-baseline comparison semantics
- CSI consumer artifact manifest
- CSI consumer minimal reproduction

## Headline Results

- `total_consumers_audited = 15`
- `raw_only_high_priority_paths = 0`
- `all_consumers_accept_csi = true`
- `same_csi_object_used_for_all_methods = true`
- `native_receiver_success = true`
- `teacher_used_during_inference = false`
- `no_new_fallback_introduced = true`

## Important Caveat

- unified-vs-baseline is a `cross_run_comparison`
- `same_csi_tensor_signature = false`
- `strict_equivalence_claim_allowed = false`
- strict same-batch equivalence was already validated in `v0.6.0`
- this `v0.7.0` comparison should not be interpreted as a strict equivalence test

## Known Limitations

- not a full native-only benchmark
- precoder / learned model interface remains project-side bridge
- no Sionna RT
- no ray tracing
- no 5G NR full stack
- not production e2e
- no stable learned `> WMMSE-iter5` claim

## Reproduction Commands

python scripts/audit_csi_consumers.py \

python scripts/reproduce_sionna_csi_consumer_minimal.py \
